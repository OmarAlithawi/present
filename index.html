Directorate Flow

The directorate operates two parallel queues. Routine school-approved requests get a final verification pass. Stalled requests, where the school has gone silent, are handled directly. Both queues converge on a single decision node, then everything is logged and patterns are reported up to the ministry.

Edge Case

What if the school doesn’t exist?

A proposed sub-flow when a student names a school that isn’t already registered.

  1. The student enters the info.
  2. The directorate receives the request.
  3. They check if the school already exists or not.
  4. If it does, the directorate rejects the request with a reason.
  5. If not, they check if it is named correctly or not.
  6. If it is correct, then approved; else, edit request.
